<objective>
改进架构：tools.json 的读取由上层（app）负责，agent 只接收 JSON 字符串。

Purpose: 实现职责分离 - agent 负责工具执行，app 负责工具定义读取
Output: NativeMobileAgentApi 接收 JSON 字符串，不读取文件
</objective>

<context>
架构改进背景：
- tools.json 是 call_android_tool 能力的说明文档
- 应该由上层（app）提供这个定义
- agent 模块只负责"执行工具"，不关心工具定义从哪里来

当前问题：
- NativeMobileAgentApi 接收 InputStream，自己读取文件
- 职责边界模糊：agent 参与了"读取工具定义"

改进方案：
- NativeMobileAgentApi: initialize(String toolsJson, String configPath)
- NativeMobileAgentApiAdapter: 读取 tools.json，传递 JSON 字符串
</context>
