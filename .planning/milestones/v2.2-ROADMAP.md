# Milestone v2.2: App 层动态注入 Android 工具

**Status:** ✅ SHIPPED 2026-03-10
**Phases:** 1-3
**Total Plans:** 3

## Overview

实现 app 层可动态注册自定义 Tool 到 AndroidToolManager，使 app 层能够扩展 Android 能力。

## Phases

### Phase 1: Tool 注册接口

**Goal**: App 层可以通过接口注册自定义 Tool 到 AndroidToolManager，支持运行时动态添加

**Depends on**: Nothing (first phase)

**Requirements**: INJT-01, INJT-02, INJT-03

**Plans**: 1 plan

- [x] 01-tool-register/01-PLAN.md — 添加 registerTool() 接口，内置 Tool 迁移到 app 层

**Details:**
- App 层可通过 registerTool(name, description, executor) 接口注册自定义 Tool
- 注册的 Tool 可在应用运行期间动态添加，无需重启
- Tool 注册时需要提供名称、描述和执行器，三者缺一不可

---

### Phase 2: Tool 生命周期管理

**Goal**: 支持 Tool 的查询、注销，并确保变更能主动推送给 Agent（LLM）

**Depends on**: Phase 1

**Requirements**: INJT-04, INJT-05, INJT-06

**Design Decisions**:
- 批量操作优化：提供 registerTools(Map) 和 unregisterTools(List) 接口
- 主动推送机制：Tool 变更后立即调用 generateToolsJson() + NativeMobileAgentApi.setToolsJson()
- 静态声明：仅使用动态生成，assets/tools.json 不再需要

**Plans**: 1 plan

- [x] 02-tool-lifecycle/02-02-PLAN.md — 添加查询、注销和批量操作接口

**Details:**
- App 层可以查询已注册的所有 Tool 列表（getToolNames）
- App 层可以注销已注册的 Tool（支持内置和自定义 Tool）
- Tool 变更后主动推送给 Agent，LLM 能感知到新增/移除的 Tool
- 提供批量操作接口，避免频繁刷新

---

### Phase 3: 动态 Tool 调用与验证

**Goal**: Agent 能够调用通过 App 层注册的 Tool，完成示例验证

**Depends on**: Phase 2

**Requirements**: INJT-07, INJT-08, INJT-09, INJT-10, INJT-11

**Design Decisions**:
- 验证对象选择：将 SearchContactsTool 和 SendImMessageTool 从 agent-android 迁移到 app 层

**Plans**: 1 plan

- [x] 03-tool-call/03-01-PLAN.md — 迁移 SearchContactsTool 和 SendImMessageTool 到 app 层

**Details:**
- Agent 可以调用通过 App 层注册的 Tool（通过 Tool 名称）
- Tool 执行结果可以返回给 Agent（LLM）
- 自定义 Tool 与内置 Tool 使用相同的调用通道（无差异）
- SearchContactsTool 和 SendImMessageTool 从 agent-android 迁移到 app 层注册
- 验证 SearchContactsTool 和 SendImMessageTool 可以被 Agent 正常调用并返回结果

---

## Milestone Summary

**Key Decisions:**

- Decision: 使用 app 层 package 路径 com.hh.agent.tool 进行 Tool 注册
- Decision: 批量操作优化 - 提供 registerTools(Map) 和 unregisterTools(List) 接口
- Decision: 主动推送机制 - Tool 变更后立即刷新 tools.json
- Decision: 删除静态 assets/tools.json，改用动态生成

**Issues Resolved:**

- 解决 app 层无法扩展 Android 能力的问题
- 实现运行时动态注册 Tool

- (无)
