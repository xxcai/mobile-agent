<objective>
设计并定义 `call_android_tool` 通用工具的 inputSchema，让 LLM 通过这个单一管道调用 Android 功能。

Purpose: 实现"单一工具原则"，LLM 只需知道 function 名称，由 Android 端决定具体执行什么。

Output: tools.json 包含 call_android_tool 定义，function 参数使用 enum
</objective>

<context>
当前问题:
- 暴露给 LLM 多个具体工具 (show_toast, read_file, etc.)
- LLM 直接选择工具名，Android 端无控制权

解决方案:
- 只暴露一个工具: call_android_tool
- function 参数使用 enum 限制可选值
- Android 端通过注册表映射 function → Executor
</context>

创建 tools.json，定义通用工具 call_android_tool:

```json
{
  "tools": [
    {
      "name": "call_android_tool",
      "description": "调用 Android 设备功能。可用功能:\n- display_notification: 显示通知, 参数: title, content\n- show_toast: 显示Toast, 参数: message, duration\n- read_clipboard: 读取剪贴板\n- take_screenshot: 截屏",
      "parameters": {
        "type": "object",
        "properties": {
          "function": {
            "type": "string",
            "description": "要调用的功能名称",
            "enum": ["display_notification", "show_toast", "read_clipboard", "take_screenshot"]
          },
          "args": {
            "type": "object",
            "description": "功能参数，JSON 对象格式"
          }
        },
        "required": ["function", "args"]
      }
    }
  ]
}
```

关键点:
1. name 必须是 "call_android_tool"
2. function 使用 enum 限制可选功能
3. args 使用通用 object 类型
4. description 清晰列出所有可用功能
