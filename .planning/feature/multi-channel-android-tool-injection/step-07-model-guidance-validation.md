# Step 07：验证模型选通道、选工具与参数组织效果

状态：待实施
前置：Step 06 完成并确认

## 目标

用最小验证面确认新的“通道边界 + 工具自描述”方案，确实能帮助模型更稳定地：

- 选对 channel
- 选对工具
- 组织对参数

## 本步骤要做的事

- 在 `MainActivity` 或等价的 app 层测试入口中输出最终 schema
- 构造若干典型用户意图样例
- 检查这些样例对应的推荐通道和工具是否清晰

建议覆盖至少三类意图：

- 业务型：如“给李四发消息说明天开会”
- 查询型：如“查一下张三是不是联系人”
- 手势型：如“点击屏幕右下角按钮”

## 本步骤不做的事

- 不做真实 LLM 自动评测框架
- 不改真实手势运行时
- 不新增新的通道

## 测试方式

- 使用 `MainActivity` 展示：
  - 各通道最终 description
  - `call_android_tool` 的工具摘要
  - `android_gesture_tool` 的动作摘要
- 人工检查典型意图是否能映射到正确通道
- 回归 `im_sender` skill 的既有调用方式

## 验收标准

- `call_android_tool` 和 `android_gesture_tool` 的边界清晰
- 现有业务工具的用途和参数组织方式能从 schema 中直接看出
- `im_sender` skill 不需要修改即可继续工作
- 工程编译通过
