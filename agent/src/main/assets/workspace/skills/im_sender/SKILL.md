# IM 消息发送助手

当用户请求发送即时消息时，使用此 Skill 完成以下工作流程。

## 触发条件

- 用户要求"发消息给xxx"
- 用户要求"发送IM消息"
- 用户要求"给xxx发一条消息"

## 工作流程

### 步骤 1: 搜索联系人

使用 `search_contacts` 工具搜索联系人。

**输入参数：**
```json
{
  "query": "联系人姓名"
}
```

**输出示例：**
- 多个匹配项：`[{"name": "张三", "id": "001", "department": "技术部"}, {"name": "张三", "id": "002", "department": "市场部"}]`
- 单个匹配项：`[{"name": "李四", "id": "003", "department": "产品部"}]`
- 无匹配：`[]`

### 步骤 2: 处理搜索结果

根据搜索结果数量采取不同策略：

- **多个匹配项**：同时返回所有结果，让用户选择要发送给哪个人
- **单个匹配项**：直接使用该联系人，无需询问用户
- **无匹配项**：告知用户未找到联系人，请确认姓名

### 步骤 3: 发送消息

使用 `send_im_message` 工具发送消息。

**输入参数：**
```json
{
  "contact_id": "联系人ID",
  "message": "消息内容"
}
```

**输出示例：**
`{"success": true, "result": "消息已发送给 001: 你好"}`

### 步骤 4: 返回结果

向用户确认消息发送成功，包括：
- 联系人姓名和部门
- 发送的消息内容
- 发送状态

## 决策规则

根据用户锁定决策：
- 多个匹配项：同时返回所有结果，让用户选择
- 单个匹配项：直接发送，不询问

## 示例对话

**用户：** "给张三发一条消息说晚上开会"

**Agent：**
1. 搜索联系人"张三"
2. 返回搜索结果：找到2个"张三"（技术部001，市场部002）
3. 请用户选择要发送给哪一个
4. 用户选择后，调用 send_im_message 发送消息
5. 返回发送结果

---

**用户：** "给李四发一条消息说明天上午10点开会"

**Agent：**
1. 搜索联系人"李四"
2. 返回搜索结果：找到1个"李四"（产品部003）
3. 因为是唯一匹配，直接调用 send_im_message 发送消息
4. 返回发送成功结果
