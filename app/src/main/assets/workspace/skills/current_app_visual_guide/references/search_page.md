# Search Page

基于当前前台界面整理的“搜索”页结构。

## 页面主体结构

“搜索”页是当前应用中的全局检索入口，整体通常分为三层：

- 顶部搜索输入区
- 中部搜索范围切换区
- 下部历史搜索与推荐内容区

## 搜索输入区

页面顶部通常是明显的搜索输入框。

这一部分用于输入关键词，是整个搜索页最核心的交互区域。

## 搜索范围切换区

当前界面中，可见的搜索范围包括：

- 联系人
- 群组
- 聊天记录
- 文件
- 更多

这说明搜索页不是只搜一种内容，而是支持跨类型检索。

当点击“更多”后，这一层会进一步展开为更完整的搜索分类，当前可见的扩展项包括：

- IT服务
- 邮件
- 公文
- 公众号
- 社区
- 知识

展开后，右侧入口会变成“收起”，表示这一层支持在精简视图和展开视图之间切换。

## 最近搜索

搜索范围区域下方通常会展示“最近搜索”。

这部分用于承接用户最近查找过的人、词或对象，帮助快速回到高频搜索目标。

## 推荐区

页面下半部分通常会展示推荐内容，当前可见内容包括：

- 推荐快捷词
- 知识推荐
- 推荐文章或推荐内容列表

这说明搜索页除了检索入口本身，也承担了一部分内容发现和搜索引导功能。

## 页面功能概括

“搜索”页可以理解为当前应用中的统一搜索首页。
它连接了联系人、群组、聊天记录、文件等多类内容，也通过最近搜索和推荐内容帮助用户快速进入目标。
