# Contacts Page

基于当前前台界面提炼的“通讯录”页稳定结构。

## 页面主体结构

“通讯录”页当前可见结构通常包含：

- 顶部搜索入口
- 一组联系人分类入口
- “我的联系人”区域
- 组织查看或联系人管理相关入口

这说明“通讯录”页的主要职责是联系人、组织和关系管理，而不是消息处理或知识阅读。

## 常见分类入口

当前界面中，可见的联系人分类入口包括：

- 小微
- 组织
- 团队
- 关注
- 外部
- 公众号

这些入口应被视为“通讯录页内的二级分类”，不是底部一级导航。

## 我的联系人区域

当前界面中存在“我的联系人”区域。

该区域可能出现两类状态：

### 有联系人时

- 展示部分已存在的联系人
- 联系人通常以列表形式出现
- 每个联系人项通常可作为进入联系人详情、发起会话或继续操作的入口

### 无联系人或当前分类为空时

- 空状态提示
- 引导去“组织”查看联系人的说明
- “查看组织”这类明确操作按钮

无论是联系人列表还是空状态，都不改变该区域属于“我的联系人”的事实。
空状态不代表“通讯录”页不可用，只代表当前子区域暂无内容。

## 页面功能概括

“通讯录”页可以理解为当前应用中的联系人与组织入口。
它既承载直接找人的需求，也承载查看组织、团队、外部联系人和关注对象等关系型浏览需求。
