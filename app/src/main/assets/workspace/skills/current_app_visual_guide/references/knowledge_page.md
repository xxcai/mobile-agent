# Knowledge Page

基于当前前台界面提炼的“知识”页稳定结构。

## 页面主体结构

“知识”页当前可见结构更像内容分发页，通常包含：

- 顶部搜索入口
- 上方二级切换区，例如“推荐”“订阅”
- 纵向内容流
- 内容卡片中的封面、标题、日期等元信息

这说明“知识”页的主要职责是内容浏览、订阅和阅读，而不是消息处理或业务应用启动。

## 内容卡片特征

当前界面中，可见内容通常表现为连续堆叠的内容项，常带有这些信息：

- 封面图或内容配图
- 标题
- 日期或发布时间

这类内容应被视为“文章 / 资讯 / 知识内容入口”，而不是业务应用入口。

## 页面功能概括

“知识”页可以理解为当前应用中的内容分发与阅读入口。
它强调推荐内容、订阅内容和文章浏览，整体更接近资讯首页或知识门户。
