# App Map

基于当前应用前台界面归纳的一级结构。

## 一级 Tab

当前应用底部存在稳定的一级导航栏，其下可见五个一级 Tab：

| Tab 文案 | 识别锚点 |
|---|---|
| 消息 | 底部导航文案或图标语义为“消息” |
| 邮件 | 底部导航文案或图标语义为“邮件” |
| 通讯录 | 底部导航文案或图标语义为“通讯录” |
| 业务 | 底部导航文案或图标语义为“业务” |
| 知识 | 底部导航文案或图标语义为“知识” |

## 一级区域概览

### 消息

- 主要承载即时沟通相关内容
- 页面通常由搜索区、消息分类区和会话列表组成
- 适合承载聊天、通知、会话浏览等功能

### 邮件

- 作为独立一级区域存在
- 主要承载邮件收发、阅读、筛选和搜索
- 页面通常包含邮件筛选入口、搜索入口和可滚动的邮件列表
- 详细结构见 `mail_page.md`

### 业务

- 主要承载业务应用入口和工作场景能力
- 页面常见搜索、最近使用、应用宫格和业务卡片
- 更接近应用广场或工作台

### 知识

- 主要承载资讯、文章、订阅内容和阅读场景
- 页面常见搜索入口、内容切换区和纵向内容流
- 更接近内容分发或知识阅读首页

### 通讯录

- 主要承载联系人、组织、团队和关系管理
- 页面常见搜索入口、联系人分类入口和联系人区域
- 更接近联系人导航与组织浏览入口

## 通用页面骨架

当前应用主界面通常具备这些稳定区域：

- 顶部标题栏
- 中间主内容区
- 底部一级导航栏

这三个区域构成了当前应用大多数一级页面的共同骨架。

## 一级 Tab 之外的常见页面

### 个人中心

- 个人中心是当前应用中的个人信息与个人服务聚合页
- 页面上半部分通常是头像、姓名、工号等个人信息区域
- 页面下半部分通常是按分组堆叠的功能入口宫格
- 常见分组包括常用、HR服务、行政服务、其他等
- 这一页更接近“个人服务首页”，而不是底部一级 Tab

### 搜索页

- 搜索页是当前应用中的跨模块检索入口
- 页面上方通常是搜索输入框
- 页面中部通常是搜索范围或搜索类型切换区
- 页面下方通常会展示最近搜索、推荐词或推荐内容
- 这一页更接近全局搜索首页，而不是某个单独业务模块

### 主界面快捷操作弹层

- 这是主界面上展开的菜单型浮层
- 通常从右上区域弹出
- 以纵向列表形式承载若干快捷动作
- 更接近“快速操作入口集合”，而不是独立页面
