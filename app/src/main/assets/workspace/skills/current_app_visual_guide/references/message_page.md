# Message Page

基于当前前台界面提炼的“消息”页稳定结构。

## 顶部区域

- 顶部有明确标题栏
- 左上存在返回或侧边入口的点击区域
- 右上至少有两个可点击图标区域

## 搜索区

消息页顶部内容区下方有稳定搜索入口：

- 搜索输入入口或搜索提示
- 右侧帮助或说明入口

## 二级聚类条

消息页搜索区下方存在一组会话聚类入口：

- 表现为一排可切换的消息筛选项

当前界面中可见的子项：

- 我
- 稍后
- 未读
- 特别关注

这些子项是“消息”页内的二级筛选，不是一级 Tab。

## 会话列表

会话列表主要锚点：

- 列表中的会话名称
- 会话时间
- 最近一条摘要
- 可点击的会话行

## 页面功能概括

“消息”页可以理解为当前应用中的沟通首页。
它聚合了消息搜索、消息分类和最近会话，是聊天、通知和消息浏览最集中的区域。
