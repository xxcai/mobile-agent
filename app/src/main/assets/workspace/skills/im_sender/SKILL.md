# IM 消息发送助手

帮助用户通过自然语言发送即时消息，自动处理联系人搜索、匹配和消息发送的全流程。

## 触发条件

此技能应该在以下情况下触发：

- 用户要求"发消息给xxx"
- 用户要求"给xxx发消息说..."
- 用户要求"告诉xxx..."
- 用户要求"通知xxx"
- 用户要求"提醒xxx"
- 用户要求"联系xxx"
- 用户说"让xxx..."（传递信息意图）
- 用户说"跟xxx说..."（传递信息意图）
- 用户说"叫xxx..."（传递信息意图）

**注意**: 即使用户没有明确说"发送消息"，只要意图是向某人传递信息，就应使用此技能。

## 工作流程

### 步骤 1: 提取信息

从用户请求中提取：
- **联系人姓名/关键字**: 必需
- **消息内容**: 如未提供，后续询问

### 步骤 2: 搜索联系人

调用 `call_android_tool` 工具，function 为 `search_contacts`。

**完整调用格式**:
```json
{
  "function": "search_contacts",
  "args": {
    "query": "联系人姓名或关键字"
  }
}
```

**返回值解析**:
| 返回 | 含义 | 后续动作 |
|------|------|---------|
| `{"success": true, "result": []}` | 无匹配 | 告知用户，建议确认姓名 |
| `{"success": true, "result": [1项]}` | 单一匹配 | 直接使用该联系人 |
| `{"success": true, "result": [多项]}` | 多个匹配 | 列出选项让用户选择 |
| `{"success": false, "error": ...}` | 调用失败 | 检查错误，向用户解释 |

### 步骤 3: 处理搜索结果

**多个匹配项**:
```
找到 {N} 位联系人：
1. {name} ({department})
2. {name} ({department})
...
请问您想发送给哪一位？
```

**无匹配项**:
```
未找到名为"{query}"的联系人，请确认姓名是否正确。
```

### 步骤 4: 确认消息内容

如果用户尚未提供消息内容：
```
请问您想发送什么消息？
```

### 步骤 5: 发送消息

调用 `call_android_tool` 工具，function 为 `send_im_message`。

**完整调用格式**:
```json
{
  "function": "send_im_message",
  "args": {
    "contact_id": "联系人ID（从搜索结果获取）",
    "message": "消息内容"
  }
}
```

**返回值解析**:
| 返回 | 含义 | 后续动作 |
|------|------|---------|
| `{"success": true, "result": "..."}` | 发送成功 | 向用户确认 |
| `{"success": false, "error": "..."}` | 发送失败 | 解释原因，建议重试 |

### 步骤 6: 返回结果

**成功**:
```
消息已发送给 {name}（{department}）: "{message}"
```

**失败**:
```
消息发送失败：{错误原因}。请稍后重试。
```

## 决策规则

| 条件 | 决策 | 原因 |
|------|------|------|
| 单一联系人匹配 | 直接发送，无需确认 | 避免不必要的交互延迟，提升用户体验；唯一匹配时确认是多余的 |
| 多个联系人匹配 | 列出选项，让用户选择 | 防止发送给错误的人，同名联系人很常见 |
| 无联系人匹配 | 不调用发送工具，提示用户 | 避免向无效ID发送消息导致错误 |
| 消息内容未提供 | 先询问消息内容再发送 | 发送空消息没有意义，且可能造成困惑 |
| 用户取消发送 | 确认取消并结束流程 | 尊重用户意愿，不强制执行 |
| 消息过长（>500字） | 提示可能被截断，询问是否继续 | IM平台通常有消息长度限制 |

## 边界情况处理

### 用户取消发送
```
用户: 算了，不发了
Agent: 好的，已取消发送。
```

### 消息过长
```
用户: 给张三发一个很长的消息...（超过500字）
Agent: 消息内容较长，可能会被截断。是否继续发送？
```

### 重复发送相同消息
```
用户: 给张三发消息说开会
Agent: （发送成功）
用户: 再发一遍
Agent: 您确定要再次发送相同内容给张三吗？
```

### 网络错误
```
Agent: 消息发送失败：网络连接异常。请检查网络后重试。
```

## 错误处理

### 工具调用失败

当工具返回 `{"success": false, ...}` 时：

1. **检查 `error` 字段**确定失败原因
2. **常见错误类型**:
   - `missing_required_param`: 缺少必需参数，提示用户提供
   - `tool_not_found`: 工具不存在，这是系统问题，向用户致歉
   - `execution_failed`: 执行失败，检查 `message` 字段获取详情

3. **向用户解释时**:
   - 使用简单语言，避免技术术语
   - 提供可行的解决方案
   - 如果问题持续，建议稍后重试

### 错误响应模板

| 错误类型 | 用户提示 |
|---------|---------|
| 缺少参数 | "请提供{参数名}信息" |
| 联系人不存在 | "未找到该联系人，请确认姓名" |
| 网络错误 | "网络连接异常，请稍后重试" |
| 权限问题 | "发送消息需要相关权限，请检查设置" |
| 未知错误 | "发送失败，请稍后重试" |

## 示例对话

### 完整流程（多匹配项）

**用户**: 给张三发消息说明天开会

**Agent**: 
1. 调用 `call_android_tool({"function": "search_contacts", "args": {"query": "张三"}})`
2. 收到 2 个结果
3. "找到2位联系人：1. 张三(技术部) 2. 张三(市场部)。请问您想发送给哪一位？"

**用户**: 技术部

**Agent**: 
1. 使用ID "001"
2. 调用 `call_android_tool({"function": "send_im_message", "args": {"contact_id": "001", "message": "明天开会"}})`
3. "消息已发送给技术部的张三：明天开会"

4. "消息已发送给产品部的李四：下午3点开会"

### 无匹配项

**用户**: 给王五发消息

**Agent**: 
1. 调用 `call_android_tool({"function": "search_contacts", "args": {"query": "王五"}})`
2. 收到空结果
3. "未找到名为'王五'的联系人，请确认姓名是否正确"

---

### 消息内容未提供

**用户**: 给李四发个消息

**Agent**: 
1. 调用 `call_android_tool({"function": "search_contacts", "args": {"query": "李四"}})`
2. 收到 1 个结果
3. "找到联系人：李四(产品部)。请问您想发送什么消息？"

**用户**: 说明天下午3点开会

**Agent**: 
1. 调用 `call_android_tool({"function": "send_im_message", "args": {"contact_id": "003", "message": "明天下午3点开会"}})`
2. "消息已发送给产品部的李四：明天下午3点开会"

---

### 隐式触发（非明确"发消息"表达）

**用户**: 让张三来我办公室一趟

**Agent**: 
1. 识别传递信息意图，触发此技能
2. 调用 `call_android_tool({"function": "search_contacts", "args": {"query": "张三"}})`
3. 找到联系人后发送消息："请来我办公室一趟"

---

### 用户取消

**用户**: 给李四发消息

**Agent**: "找到联系人：李四(产品部)。请问您想发送什么消息？"

**用户**: 算了不发了

**Agent**: "好的，已取消发送。"
