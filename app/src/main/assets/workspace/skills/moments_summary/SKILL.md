# 朋友圈内容总结助手

帮助用户基于当前页面视图读取朋友圈 feed，并输出简洁、可信、带边界说明的总结。

## 触发条件

此技能应该在以下情况下触发：

- 用户要求“总结朋友圈”
- 用户要求“看看朋友圈都发了什么”
- 用户要求“概括一下发现页内容”
- 用户要求“总结这页动态”
- 用户要求“看看大家最近在发什么”
- 用户要求“把这个 feed 总结一下”

**注意**:

- 只有当任务核心是“阅读当前 feed 内容并总结”时才触发此技能。
- 如果用户要点赞、评论、发朋友圈或打开某条动态，不要用此技能完成动作本身。

## 工作流程

### 步骤 1: 先看页面

优先调用 `android_view_context_tool`，读取当前可见页面结构。

**完整调用格式**:
```json
{
  "source": "native_xml",
  "targetHint": "朋友圈动态"
}
```

如果当前页面不是朋友圈 / 动态流，而是别的页面：

- 明确告诉用户当前页不是可总结的朋友圈 feed
- 不要凭空编造总结

### 步骤 2: 提取可见动态

从当前可见视图中提取每条动态的关键信息，尽量包括：

- **发布者**
- **时间**
- **正文**
- **位置**
- **图片/视频说明**
- **点赞/评论等统计提示**

如果某项信息看不清：

- 可以省略该字段
- 不要猜测

### 步骤 3: 判断是否需要滑动补读

默认先基于**当前可见内容**总结。

如果用户明确要求“完整一点”“多看几条”“继续往下看再总结”，或者当前可见动态明显不足以支撑总结，可以**受控向下滑动页面**，继续查看更早的朋友圈内容，再补读：

1. 调用一次 `android_gesture_tool`，使用 `action=swipe`、`direction=down`、`scope=feed`，必要时可配 `amount=medium`
2. 再次调用 `android_view_context_tool`
3. 合并新旧可见内容

**限制**:

- 最多补读 2 次
- 每次补读后都要重新看页面，不要盲滚
- 补读目标是查看**之前/更早的朋友圈动态**
- 如果滚动后没有出现新的动态，应停止继续滚动
- 这里的滚动能力依赖 gesture runtime 的真实页面事件注入，不要在 Skill 中自己编造裸坐标

### 步骤 4: 去重

如果多次读取后出现重复动态，按以下信息做近似去重：

- 发布者
- 时间
- 正文开头

不要把同一条动态重复计入总结。

### 步骤 5: 生成总结

输出时优先包含：

1. **整体主题**
2. **主要动态摘要**
3. **可见范围说明**

推荐输出结构：

```text
这页朋友圈主要在发：
1. ...
2. ...
3. ...

整体上，大家最近主要在聊/晒：
- ...
- ...

说明：以上总结基于当前可见的 N 条动态。
```

## 决策规则

| 条件 | 决策 | 原因 |
|------|------|------|
| 当前页就是朋友圈 feed | 直接读视图并总结 | 任务目标就是总结当前内容 |
| 当前页不是朋友圈 feed | 明确说明当前页不对 | 避免凭空总结 |
| 信息不完整 | 只总结看得到的部分 | 保持可信 |
| 用户要求更完整，或希望看之前的朋友圈 | 最多补读 2 次 | 允许通过滑动查看更早内容，但控制动作范围 |
| 多次读取出现重复动态 | 去重后再总结 | 避免重复计数 |
| 页面内容很少 | 明确说明样本有限 | 防止过度概括 |

## 输出要求

- 总结要**简洁、可信、可追溯到当前可见内容**
- 优先使用“这页可见内容显示”“当前看到的动态主要是”这类表述
- 不要把猜测写成事实
- 不要把未看到的内容写进总结
- 如果只看了当前屏幕，必须明确说明“基于当前可见内容”

## 错误处理

### 看不到可用内容

如果 `android_view_context_tool` 没有返回可读的 feed 内容：

```text
我当前没有在页面上看到可总结的朋友圈动态内容。你可以先打开朋友圈页，我再帮你总结。
```

### 内容过少

如果当前只看到 1 到 2 条动态：

```text
我目前只能看到少量动态，先基于这几条做简要总结：...
```

### 需要继续查看

如果用户要求“继续看”“多看一些”或“看看之前的朋友圈”：

- 先说明将继续向下滑动，查看更早的朋友圈动态
- 再执行一次受控滚动和补读

## 示例对话

### 示例 1：基于当前可见内容总结

**用户**: 帮我总结一下朋友圈都在发什么

**Agent**:
1. 调用 `android_view_context_tool({"source":"native_xml","targetHint":"朋友圈动态"})`
2. 提取当前可见动态
3. 输出：

```text
这页朋友圈主要在发：
1. 有人在分享迪士尼出游和夜间烟花体验。
2. 有人在发工作复盘，提到周报模板和测试排查。
3. 也有人在晒骑行、宠物和读书会内容。

整体上，这一页可见动态主要围绕出行休闲、工作日常和个人生活记录。

说明：以上总结基于当前可见的朋友圈内容。
```

---

### 示例 2：要求多看几条

**用户**: 再往下看一点，然后总结完整一些

**Agent**:
1. 调用 `android_gesture_tool` 向下滚动一次
2. 再次调用 `android_view_context_tool`
3. 合并并去重
4. 输出更新后的总结，并说明“基于当前屏幕 + 补读内容”

---

### 示例 3：查看之前的朋友圈

**用户**: 往前翻一点，看看之前的朋友圈都在发什么

**Agent**:
1. 说明将继续向下滑动查看更早动态
2. 调用 `android_gesture_tool` 向下滚动一次
3. 再调用 `android_view_context_tool`
4. 对新增内容和当前内容一起总结，并说明总结基于补读后的可见范围

---

### 示例 4：当前页不对

**用户**: 总结一下朋友圈

**Agent**:
```text
我当前没有看到朋友圈 feed 页面内容。你可以先打开朋友圈页，我再帮你总结。
```
