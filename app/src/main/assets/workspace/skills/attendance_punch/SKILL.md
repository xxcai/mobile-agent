# 打卡助手

帮助 Agent 在宿主 App 中完成考勤打卡。

标准路径：

1. 回到首页
2. 点击底部 Tab 中的“业务”
3. 在业务页面中找到“每日打卡”卡片
4. 进入打卡页面后点击“打卡”按钮
5. 观察打卡结果并向用户反馈

## 触发条件

以下表达通常应触发本技能：

- “打卡”
- “帮我打卡”
- “上班打卡”
- “下班打卡”
- “考勤打卡”
- “签到”

## 核心规则

### 1. 必须先点击打卡按钮，再判断是否成功

即使页面中已经存在历史打卡记录，仍然需要执行本次打卡动作。

不要因为页面已经显示“首次”“末次”或已有时间值，就在点击前直接判定任务成功。

### 2. 成功判断采用“点击前后对比”

点击打卡按钮后，成功条件分两类：

- 如果点击前没有打卡记录，点击后 `tv_attendance_magnet_signed_first_date` 出现，表示打卡成功
- 如果点击前已经有打卡记录，点击后 `tv_attendance_magnet_signed_last_date` 的值发生变化，表示打卡成功

此外，如果页面明确出现以下结果词，也可以判定成功：

- 打卡成功
- 已打卡
- 已完成打卡
- 打卡时间已更新
- 打卡时间更新
- 新的打卡时间
- 上班打卡成功
- 下班打卡成功

### 3. 不要重复点击打卡按钮

如果点击后已经满足上述任一成功条件，就停止继续点击。

### 4. 失败或阻断条件

如果页面出现以下信息，应停止重试并向用户反馈：

- 打卡失败
- 不在打卡范围
- 定位失败
- 网络异常
- 无权限
- 未到打卡时间

## 工作流程

### 步骤 1：先看当前页面

先调用 `android_view_context_tool` 判断当前页面属于哪一类：

- 首页
- 业务页
- 打卡页
- 其他页面

### 步骤 2：如不在首页，优先安全返回首页

如果当前不在首页，也不在业务页或打卡页，优先通过页面可见的返回、关闭、取消等安全入口回到首页。

不要盲目点击页面中部卡片或未知入口。

### 步骤 3：从首页进入业务页

在首页点击底部 Tab “业务”，并在点击后重新观察页面，确认已进入业务页。

### 步骤 4：从业务页进入打卡页

在业务页中优先寻找标题明确为“每日打卡”的卡片，优先点击“每日打卡”标题所在的卡片容器，而不是点击页面中任意出现的“打卡”字样。

如果当前卡片同时出现以下任一线索，不要把它当作打卡入口：

- 日历
- 日程
- 暂无日程
- 发起人
- 时间段，例如 `12:00-13:00`
- 具体日程标题，例如“干饭”

如果只看到单独的“打卡”字样，但该字样位于日历/日程卡片区域内，也不要点击。

如果首屏没有看到“每日打卡”卡片，可以向下滚动页面继续查找。这里的目标是查看更下方卡片内容，不要触发下拉刷新。

### 步骤 5：点击打卡按钮

在打卡页找到“打卡”“立即打卡”“上班打卡”“下班打卡”等按钮后点击。

点击后必须重新观察页面，并基于“点击前后字段变化”判断是否成功。

## 输出要求

### 成功

```text
已完成打卡，页面显示本次打卡已生效。
```

### 失败

```text
当前没有完成打卡，页面提示：定位失败。需要先确认定位权限或网络状态。
```

### 未找到稳定入口

```text
我没有找到稳定的打卡入口，已停止继续点击，避免误操作。
```
